# HW10-11 – компьютерное зрение в PyTorch: CNN, transfer learning, detection/segmentation

## 1. Кратко: что сделано
- **Часть A:** Выбран датасет **SVHN** (цифры с домов). Причина: малый размер изображений (32x32) для минимизации нагрузки на CPU, при этом задача нетривиальна для простой CNN.
- **Часть B:** Выбран трек **Detection** на датасете **Pascal VOC 2012**. Причина: универсальность и возможность наглядно показать влияние порога уверенности (threshold) на метрики.
- **Сравнение:** В части A сравнивались простая CNN, аугментации и Transfer Learning (ResNet18). В части B сравнивались два порога детекции (0.3 и 0.7).

## 2. Среда и воспроизводимость
- **Python:** 3.8+
- **torch / torchvision:** Latest stable
- **Устройство:** CPU / CUDA (автоопределение)
- **Seed:** 42
- **Запуск:** Открыть `HW10-11.ipynb` и выполнить Run All.

## 3. Данные
### 3.1. Часть A: классификация
- **Датасет:** SVHN
- **Разделение:** Train (80%) / Val (20%) / Test (official test split)
- **Transforms:** Normalization, RandomHorizontalFlip, ColorJitter (для аугментаций).
- **Комментарий:** 10 классов, изображения 32x32. Задача проще, чем CIFAR100, но требует инвариантности к освещению и поворотам.

### 3.2. Часть B: structured vision
- **Датасет:** Pascal VOC 2012 (val subset)
- **Трек:** Detection
- **Ground Truth:** Bounding boxes из аннотаций VOC.
- **Предсказания:** Boxes от FasterRCNN_ResNet50_FPN.
- **Комментарий:** Использовано 20 изображений для расчета метрик в рамках демо, чтобы ускорить выполнение.

## 4. Часть A: модели и обучение (C1-C4)
- **C1:** SimpleCNN (2 conv layers) без аугментаций.
- **C2:** SimpleCNN с аугментациями (Flip, Color).
- **C3:** ResNet18 (pretrained), замороженный backbone, обучается только FC.
- **C4:** ResNet18 (pretrained), Fine-tune (layer4 + FC).
- **Loss:** CrossEntropyLoss
- **Optimizer:** Adam (lr=0.001)
- **Batch size:** 64 (CPU) / 128 (GPU)
- **Epochs:** 5-10
- **Критерий:** Best Val Accuracy.

## 5. Часть B: постановка задачи и режимы оценки (V1-V2)
- **Модель:** FasterRCNN_ResNet50_FPN (pretrained)
- **V1:** `score_threshold = 0.3` (больше предсказаний, выше Recall, ниже Precision).
- **V2:** `score_threshold = 0.7` (меньше предсказаний, выше Precision, ниже Recall).
- **IoU:** Считался между Pred Box и GT Box. Match при IoU >= 0.5.
- **Метрики:** Precision, Recall, Mean IoU для совпавших боксов.

**Сводка:**

- Аугментации (C2 vs C1): Дали прирост устойчивости.
- Transfer Learning (C3/C4): Значительный прирост скорости сходимости и качества.
- V1 vs V2: При повышении порога Precision растет, Recall падает.

## 7. Анализ
Простая CNN (C1) быстро переобучается на SVHN из-за малой емкости. Аугментации (C2) помогают сгенерализовать лучше. Pretrained ResNet (C3) сразу показывает высокий результат благодаря признакам ImageNet. Fine-tuning (C4) позволяет адаптировать признаки под специфику цифр, давая лучший результат.
В детекции порог 0.3 ловит больше объектов, но добавляет шум (False Positives). Порог 0.7 очищает шум, но пропускает сложные объекты (False Negatives). Метрика IoU корректно отражает качество локализации.

## 8. Итоговый вывод
Для классификации базовым конфигом стоит брать ResNet18 с Fine-tuning. Transfer learning критически важен при малом количестве данных или эпох. В детекции выбор порога — это компромисс между Precision и Recall, зависящий от задачи (безопасность vs комфорт).
