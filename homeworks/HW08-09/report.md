# HW08-09 – PyTorch MLP: регуляризация и оптимизация обучения

## 1. Кратко: что сделано

- **Датасет:** Выбран EMNIST (balanced) — альтернатива KMNIST, так как при загрузке KMNIST возникли проблемы с сетью. EMNIST имеет схожую структуру (28×28, grayscale), но 47 классов вместо 10, что делает задачу сложнее и интереснее для анализа регуляризации.
- **Часть A (регуляризация):** Сравнивались 4 конфигурации — базовый MLP, MLP с Dropout, MLP с BatchNorm, и лучшая из них с EarlyStopping.
- **Часть B (оптимизация):** Исследовалось влияние learning rate (слишком большой/маленький) и сравнение Adam vs SGD+momentum+weight_decay.

## 2. Среда и воспроизводимость

- **Python:** 3.10+
- **torch / torchvision:** 2.0+ / 0.15+
- **Устройство (CPU/GPU):** CUDA (если доступно) / CPU
- **Seed:** 42 (фиксирован для всех экспериментов)
- **Как запустить:** открыть `HW08-09.ipynb` и выполнить Run All.

## 3. Данные

- **Датасет:** EMNIST (split="balanced")
- **Разделение:** train/val/test — из оригинального train выделено 15% на валидацию (random_split с фиксированным seed), test оставлен как есть из torchvision.
- **Трансформации (transform):** ToTensor(), Normalize(mean=0.5, std=0.5)
- **Комментарий:** 47 классов (буквы и цифры), 28×28 пикселей, ~112,000 train изображений. Задача сложнее KMNIST из-за большего числа классов и похожих символов (например, 'l', '1', 'I').

## 4. Базовая модель и обучение

- **Модель MLP (кратко):** 2 скрытых слоя (256, 128 нейронов), активация ReLU, вход 784 (28×28 flattened), выход 47 классов.
- **Loss:** CrossEntropyLoss
- **Базовый Optimizer (для части A):** Adam (lr=1e-3)
- **Batch size:** 128 (GPU) / 64 (CPU)
- **Epochs (макс):** 30 (для E1-E3), 50 (для E4 с EarlyStopping)
- **EarlyStopping:** patience=5, metric=val_loss, min_delta=0.0005

## 5. Часть A (S08): регуляризация (E1-E4)

| Эксперимент | Конфигурация |
|-------------|--------------|
| **E1 (base)** | 2 скрытых слоя (256, 128), без Dropout/BatchNorm |
| **E2 (Dropout)** | Как E1 + Dropout(p=0.3) после каждого скрытого слоя |
| **E3 (BatchNorm)** | Как E1 + BatchNorm1d между Linear и ReLU |
| **E4 (EarlyStopping)** | Лучший из E2/E3 по val_accuracy + EarlyStopping(patience=5) |

## 6. Часть B (S09): LR, оптимизаторы, weight decay (O1-O3)

| Эксперимент | Конфигурация |
|-------------|--------------|
| **O1** | Adam, lr=1e-1 (слишком большой) — 8 эпох |
| **O2** | Adam, lr=1e-5 (слишком маленький) — 8 эпох |
| **O3** | SGD+momentum(0.9) + weight_decay=1e-4, lr=1e-2 — 15 эпох |

## 8. Анализ

На графиках E1 (baseline) виден классический признак переобучения: train loss продолжает падать, а val loss начинает расти после ~10-15 эпохи. Добавление Dropout (E2) сокращает разрыв между train и val кривыми, что указывает на лучшее обобщение. BatchNorm (E3) стабилизирует обучение — градиенты становятся более предсказуемыми, сходимость быстрее.

EarlyStopping (E4) остановил обучение на эпохе ~15-25 (в зависимости от конфигурации), когда val_loss перестал улучшаться. Это предотвратило переобучение и сохранило веса лучшей модели.

Эксперименты O1 и O2 демонстрируют критическую важность learning rate. При lr=1e-1 (O1) loss скачет или расходится — оптимизатор делает слишком большие шаги. При lr=1e-5 (O2) модель практически не обучается — шаги слишком малы для прогресса за 8 эпох.

SGD+momentum+weight_decay (O3) показал себя как достойная альтернатива Adam. Momentum помогает преодолевать локальные минимумы, а weight decay действует как L2-регуляризация, ограничивая веса и улучшая обобщение.

Выбранный конфиг (Dropout+BatchNorm+EarlyStopping) разумен для EMNIST: много классов + похожие символы требуют хорошей регуляризации для предотвращения запоминания шума.

## 9. Итоговый вывод

**Базовый конфиг для будущих экспериментов:**
- MLP (256, 128) + ReLU + BatchNorm + Dropout(0.3)
- Optimizer: Adam, lr=1e-3
- EarlyStopping: patience=5, monitor=val_loss
- Batch size: 128, Epochs: до 50 с ранней остановкой

**Что можно улучшить дальше:**
1. Learning Rate Scheduler — попробовать ReduceLROnPlateau или CosineAnnealing для более тонкой настройки LR в процессе обучения.
2. Data Augmentation — для EMNIST можно добавить небольшие сдвиги/повороты изображений, чтобы улучшить обобщение.
